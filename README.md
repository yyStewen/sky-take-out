# 苍穹外卖项目进程

## Day1
版本控制， Git仓库的创建，通过MD5加密提升登录功能安全性。
导入接口文档以及利用swagger/knife4j生成接口文档。

## Day2
实现新增员工，员工信息的分页查询，启用和禁用员工账号，员工信息修改，并导入了分类模块代码

## Day3
菜品管理功能的实现。添加菜品，分页查询，新增和修改，起售停售菜品。

## Day4
实战：完成套餐管理的全部内容，并修复和完善菜品停售起售的逻辑。

## Day5
学习redis数据库，并使用redis数据库，以及spring data redis操作redis实现设置商铺的营业状态。

## Day6
微信小程序开发。导入微信小程序代码，并完成登录功能。导入商品浏览功能代码。

## Day7
缓存菜品和套餐数据，并完成购物车模块功能。

## Day8
导入地址簿功能代码，并实现订单管理模块开发，导入了微信支付功能代码。

## Day9
项目实战：用户端历史订单模块，商家端订单管理模块以及在用户下单加入校验距离过远的逻辑。
